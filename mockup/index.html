claudex / remote-control 2026 · v0 product design

A remote control for the claude running on your own machine.

claudex runs a small service on any machine that has the Claude Code CLI installed, and exposes a polished browser interface — designed mobile-first — that mirrors the Desktop Code tab. Review diffs from the train, approve permissions from the kitchen, watch a long refactor from bed.

Found it — lib/date.ts:12 uses Intl.DateTimeFormat with the user's locale, which is different on the server (defaults to en-US) versus the client (follows navigator.language). Three options, ranked:
  1. 1.Wrap the <time> in a client-only component. Safe, but ships JS.
  2. 2.Pin the initial render locale to "en-US", then upgrade on mount. (My pick — zero JS if the locale matches.)
  3. 3.Format dates in an RSC and pass the string down. Cleanest but loses client reactivity.
Go with option 2. Pin to "en-US" for the initial render.
